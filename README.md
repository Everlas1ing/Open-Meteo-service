# Прогноз опадів на основі ML (Open-Meteo) 🌦️

Цей проєкт — прототип міні-сервісу для прогнозування погоди (опадів) за допомогою машинного навчання. Застосунок розроблено з використанням фреймворку **Streamlit**. Він автоматично завантажує реальні метеодані, навчає модель та дозволяє робити швидкі прогнози.

## 1. Які дані використовуються
Дані отримуються через HTTP-запити з відкритого API **Open-Meteo** (Historical Archive API). 
Для навчання моделі збираються щоденні (daily) показники:
- `temperature_2m_max` (Максимальна температура, °C)
- `temperature_2m_min` (Мінімальна температура, °C)
- `wind_speed_10m_max` (Максимальна швидкість вітру, км/год)
- `precipitation_sum` (Сума опадів, мм) 
- `rain_sum` (Сума дощу, мм)

**Формування цільової змінної:**
Дані про опади (`precipitation_sum`) використовуються виключно для створення цільової змінної (label). Усі значення `precipitation_sum = 0` перетворюються на клас `0` (опадів немає), а значення `precipitation_sum > 0` перетворюються на клас `1` (опади є). Завантажений набір даних автоматично зберігається у локальний файл `weather_daily.csv` для подальшого використання.

## 2. Яку модель використано
Для вирішення задачі класифікації (опади очікуються / опадів не очікується) використовується алгоритм **Random Forest Classifier** (Випадковий ліс) з бібліотеки `scikit-learn`. 

- **Ознаки (X):** `temp_max`, `temp_min`, `wind_speed`. *(Змінні опадів навмисно виключені з ознак, щоб уникнути витоку даних).*
- **Цільова змінна (y):** Бінарна класифікація (0 - немає опадів, 1 - є опади).
- **Розбиття даних:** Дані діляться на тренувальну та тестову вибірки у пропорції 80% на 20%.

Для оцінки якості моделі використовуються наступні метрики: Accuracy (Точність), Precision, Recall та F1-Score.

## 3. Структура та організація проєкту
Проєкт складається з таких файлів:

1. **`app.py`** — головний скрипт вебзастосунку Streamlit. У ньому реалізована вся логіка: 
   - формування UI (інтерфейсу);
   - надсилання параметризованих запитів до API Open-Meteo;
   - очищення та підготовка даних за допомогою `pandas`;
   - навчання ML-моделі `RandomForestClassifier`;
   - логіка видачі передбачення на основі введених користувачем даних.
2. **`requirements.txt`** — файл із переліком необхідних Python-бібліотек для запуску проєкту (streamlit, pandas, requests, scikit-learn).
3. **`weather_daily.csv`** — файл із датасетом, який автоматично генерується і зберігається після успішного завантаження метеоданих через застосунок.
4. **`README.md`** — файл із документацією проєкту (цей файл).

## 4. Послідовність дій (Як працює застосунок)
1. **Завантаження даних:** Користувач у бічній панелі вводить координати міста (за замовчуванням стоїть Київ: широта 50.45, довгота 30.52) та обирає період дат. Після натискання кнопки *"Отримати дані з Open-Meteo"*, програма робить запит до API, обробляє отриманий JSON, форматує його у DataFrame та зберігає у файл `weather_daily.csv`.
2. **Навчання моделі:** У головному вікні користувач натискає кнопку *"Навчити модель"*. Програма формує цільову змінну, розбиває дані на `train` та `test`, тренує класифікатор і відразу виводить на екран метрики якості моделі.
3. **Прогноз:** Користувач вводить довільні або очікувані погодні умови (максимальну/мінімальну температуру та швидкість вітру) у відповідні поля форми і натискає *"Зробити прогноз"*. Модель аналізує ці дані та виводить результат: *"Очікуються опади"* або *"Опадів не очікується"*, а також показує ймовірність їх виникнення у відсотках.
